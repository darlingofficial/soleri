La conoscenza dovrebbe accumularsi. Ecco come.

Livelli di Vault, cervello e trasporto funzionano su un unico motore. Un processo, zero lock-in — cambia trasporti, aggiungi conoscenza, cambia fornitori.

Agente

La configurazione del tuo agente — agent.yaml definisce identità, voce, collegamenti di dominio. Crea in un comando, personalizza in qualsiasi momento.

Domini

Domini di conoscenza integrabili. Il motore carica i domini pertinenti in base alla struttura di archiviazione del tuo agente.

Motore

Singolo processo server MCP. Vault (conoscenza), Cervello (apprendimento), Memoria (contesto), Pianificazione (flussi di lavoro). Il tuo agente lo utilizza. Aggiornamenti tramite npm update.

Trasporti

Adattatori di protocollo isolati dal nucleo: mcp.ts è disponibile ora, rest.ts e lsp.ts si integrano per VS Code, Cursor, Zed e dashboard personalizzati. Aggiungere un trasporto = un file adattatore.

Conoscenza che si accumula

Senza un vault, ogni sessione riparte da zero. Ogni pattern, antipattern e decisione che prendi va in un archivio ricercabile. Fai una domanda — ottieni la risposta del tuo team, non una generica.

Pattern, antipattern e memorie delle sessioni — tutti ricercabili.

Diventa più acuto ad ogni sessione

La curazione manuale della conoscenza non scala. Il cervello traccia quali pattern applichi, con quale frequenza e con quanta efficacia. La fiducia cresce automaticamente — nessun tag manuale necessario.

Cattura automatica, nessun tag manuale.

Conoscenza che cresce — e rimane affilata

I pattern fluiscono attraverso un ciclo di vita in quattro fasi. Il motore osserva il tuo lavoro, cattura ciò che rimane e lo accumula nel tempo.

1
Cattura Il motore nota correzioni ripetute, suggerisce pattern
↓
2
Conserva Pattern, antipattern e memorie delle sessioni entrano nell'archivio
↓
3
Rafforza I punteggi di fiducia tracciano quanto spesso i pattern hanno successo
↓
4
Compone I pattern ad alta fiducia emergono per primi, quelli deboli sono segnalati

Il tuo archivio non decade — si affina. Il cervello traccia ciò che funziona.

Non bloccato a nessun editor

La tua conoscenza non dovrebbe essere intrappolata in un solo strumento. Il motore centrale non ha dipendenze di protocollo. Gli adattatori di trasporto sono file separati — cambiali o aggiungili senza toccare il motore.

Il nucleo ha zero dipendenze di protocollo.

engine/transports/ # un file per protocollo ├── mcp.ts # Claude Code, Cursor — disponibile ora ├── rest.ts # dashboard, API personalizzate └── lsp.ts # estensioni per VS Code, Cursor, Zed engine/core/ # zero importazioni di trasporto ├── vault.ts ├── brain.ts └── memory.ts
# agent.yaml — Configurazione LLM llm: provider: anthropic # oppure: openai, ollama, nessuno model: claude-sonnet-4-6 fallback: nessuno # funziona senza chiavi API

Funziona senza chiavi API

L'infrastruttura della conoscenza non dovrebbe richiedere un abbonamento. La ricerca nell'archivio, la corrispondenza dei pattern e il tracciamento del cervello funzionano localmente. Il provider LLM è opzionale — aggiungine uno quando desideri suggerimenti AI.

Funzionalità di base garantita senza chiavi.
